# BroSDK MCP Bridge Protocol

Version: `1.0.0`

This document defines the protocol between the BroSDK browser extension and the
local MCP bridge service. The extension is the client. The MCP bridge service is
the server.

## Base URL

The extension must connect to this fixed base URL:

```text
http://mcp.brosdk.internal
```

BroSDK Browser is responsible for resolving this internal host to the real MCP
bridge address. The extension must not scan local ports, expose a user-editable
server address, or connect to arbitrary remote hosts.

The WebSocket endpoint derived from the same host is:

```text
ws://mcp.brosdk.internal/extension/ws
```

## Transport

The preferred transport is WebSocket. HTTP long polling is the compatibility
fallback.

All messages use UTF-8 JSON. Field names use `camelCase`. Unknown fields must be
ignored by receivers so the protocol can evolve without breaking old clients.

### WebSocket

The extension connects to:

```http
GET /extension/ws
```

After the socket opens, the extension sends a `hello` message and then publishes
a fresh `state` snapshot.

Client to server:

```json
{
  "type": "hello",
  "protocolVersion": "1.0.0",
  "browserId": "9f4f5b24-1d31-49a1-8f2d-130f55a4b5d6",
  "client": "brosdk-mcp-bridge",
  "capabilities": [
    "tabs",
    "windows",
    "tabGroups",
    "bookmarks",
    "history"
  ]
}
```

Server to client command:

```json
{
  "type": "command",
  "command": {
    "id": "cmd_01HZY2YTR6VAK",
    "type": "tabs.create",
    "payload": {
      "url": "https://example.com",
      "active": true
    }
  }
}
```

Client to server command result:

```json
{
  "type": "commandResult",
  "commandId": "cmd_01HZY2YTR6VAK",
  "ok": true,
  "result": {
    "tabId": 1001
  }
}
```

Client to server error result:

```json
{
  "type": "commandResult",
  "commandId": "cmd_01HZY2YTR6VAK",
  "ok": false,
  "error": {
    "code": "INVALID_ARGUMENT",
    "message": "payload.url must be a valid URL"
  }
}
```

The server may send `ping`, `pong`, `hello`, `health`, and `error` messages. The
extension should ignore message types it does not understand.

### HTTP Fallback

The extension may use the HTTP endpoints below when the WebSocket connection is
not open.

#### Health

```http
GET /extension/health
```

Response:

```json
{
  "ok": true,
  "service": "brosdk-mcp-bridge",
  "protocolVersion": "1.0.0",
  "serverTime": "2026-07-09T01:30:00.000Z"
}
```

#### Hello

```http
POST /extension/hello
Content-Type: application/json
```

Request:

```json
{
  "protocolVersion": "1.0.0",
  "browserId": "9f4f5b24-1d31-49a1-8f2d-130f55a4b5d6",
  "client": "brosdk-mcp-bridge",
  "capabilities": [
    "tabs",
    "windows",
    "tabGroups",
    "bookmarks",
    "history"
  ]
}
```

Response:

```json
{
  "ok": true
}
```

#### Publish State

```http
POST /extension/state
Content-Type: application/json
```

Request body is a `BrowserStateSnapshot`.

Response:

```json
{
  "ok": true
}
```

#### Poll Command

```http
GET /extension/commands?timeoutMs=25000
```

Response with a command:

```json
{
  "command": {
    "id": "cmd_01HZY2YTR6VAK",
    "type": "tabs.activate",
    "payload": {
      "tabId": 1001
    }
  }
}
```

Response without a command:

```json
{
  "command": null
}
```

#### Post Command Result

```http
POST /extension/commands/{commandId}/result
Content-Type: application/json
```

Success request:

```json
{
  "ok": true,
  "result": {}
}
```

Failure request:

```json
{
  "ok": false,
  "error": {
    "code": "COMMAND_FAILED",
    "message": "No tab with id 1001"
  }
}
```

Response:

```json
{
  "ok": true
}
```

## Browser Identity

`browserId` is a stable UUID generated by the extension and stored in the current
BroSDK browser profile. It identifies one extension installation inside one
browser profile.

The extension should reuse the same `browserId` across restarts of the same
profile and generate a new one only when no stored value exists.

Optional profile metadata may be included when BroSDK exposes it:

```json
{
  "profile": {
    "profileId": "profile_123",
    "name": "Research Profile"
  }
}
```

Profile metadata is optional. The extension must avoid sending secrets, proxy
passwords, cookies, local storage values, or fingerprint configuration details.

## State Snapshot

The extension publishes the complete current browser UI state, not deltas.

```json
{
  "browserId": "9f4f5b24-1d31-49a1-8f2d-130f55a4b5d6",
  "protocolVersion": "1.0.0",
  "sequence": 42,
  "timestamp": "2026-07-09T01:30:00.000Z",
  "tabs": [],
  "windows": [],
  "groups": []
}
```

`sequence` is a monotonically increasing integer scoped to the extension service
worker lifetime. The server should treat the latest received snapshot as the
authoritative state for the `browserId`.

### Tab

```json
{
  "tabId": 1001,
  "targetId": "B1C2D3E4F5",
  "windowId": 2001,
  "groupId": 3001,
  "index": 0,
  "url": "https://example.com/",
  "title": "Example Domain",
  "active": true,
  "pinned": false,
  "hidden": false,
  "status": "complete"
}
```

Fields:

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `tabId` | number | yes | Browser tab id. |
| `targetId` | string | no | CDP target id when available. |
| `windowId` | number | yes | Owning window id. |
| `groupId` | number | no | Present only when the tab is grouped. |
| `index` | number | yes | Zero-based tab index inside the window. |
| `url` | string | yes | Current visible tab URL. |
| `title` | string | yes | Current tab title. |
| `active` | boolean | yes | Whether the tab is active in its window. |
| `pinned` | boolean | yes | Whether the tab is pinned. |
| `hidden` | boolean | yes | Whether the tab is hidden by the browser. |
| `status` | string | yes | `loading` or `complete`. |

### Window

```json
{
  "windowId": 2001,
  "type": "normal",
  "focused": true,
  "state": "normal",
  "tabCount": 3,
  "activeTabId": 1001,
  "bounds": {
    "left": 0,
    "top": 0,
    "width": 1280,
    "height": 800,
    "windowState": "normal"
  }
}
```

Fields:

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `windowId` | number | yes | Browser window id. |
| `type` | string | yes | Usually `normal`, `popup`, `panel`, `app`, or `devtools`. |
| `focused` | boolean | yes | Whether the window is focused. |
| `state` | string | yes | Usually `normal`, `minimized`, `maximized`, or `fullscreen`. |
| `tabCount` | number | yes | Number of tabs in the window. |
| `activeTabId` | number | no | Active tab id when available. |
| `bounds` | object | yes | Last known window bounds. |

### Tab Group

```json
{
  "groupId": 3001,
  "windowId": 2001,
  "title": "Work",
  "color": "blue",
  "collapsed": false,
  "tabIds": [
    1001,
    1002
  ]
}
```

Fields:

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `groupId` | number | yes | Browser tab group id. |
| `windowId` | number | yes | Owning window id. |
| `title` | string | yes | Group title. |
| `color` | string | yes | Browser group color. |
| `collapsed` | boolean | yes | Whether the group is collapsed. |
| `tabIds` | number[] | yes | Current tabs in this group. |

## Command Envelope

Commands are issued by the MCP bridge service and executed by the extension.

```json
{
  "id": "cmd_01HZY2YTR6VAK",
  "type": "tabs.create",
  "payload": {},
  "createdAt": "2026-07-09T01:30:00.000Z",
  "timeoutMs": 30000
}
```

Fields:

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `id` | string | yes | Unique command id. |
| `type` | string | yes | Command type. |
| `payload` | object | yes | Command-specific payload. Use `{}` when empty. |
| `createdAt` | string | no | ISO-8601 timestamp from the server. |
| `timeoutMs` | number | no | Suggested execution timeout. |

The extension must send exactly one result for each command id it accepts.
Command execution should be followed by a fresh state snapshot.

## Error Codes

Recommended structured error codes:

| Code | Meaning |
| --- | --- |
| `INVALID_ARGUMENT` | Command payload is missing or invalid. |
| `NOT_FOUND` | Referenced tab, window, group, bookmark, or history item does not exist. |
| `UNSUPPORTED_COMMAND` | The extension does not implement the command type. |
| `PERMISSION_DENIED` | The browser denied the requested operation. |
| `COMMAND_FAILED` | The operation failed for another browser/runtime reason. |
| `TIMEOUT` | The command exceeded its execution timeout. |
| `INTERNAL` | Unexpected extension failure. |

For compatibility, servers should also accept a plain string `error` value from
older clients, but new clients should send structured errors.

## Reconnect and Ordering

- The extension should reconnect WebSocket after close or error.
- Recommended reconnect delay is 2000 ms.
- The extension should publish a fresh state snapshot after connecting and after
  every command execution attempt.
- State snapshots are authoritative and may supersede previous snapshots.
- Command results are matched by `commandId`; command completion order may differ
  from command issue order.
- The server must not assume a command succeeded until it receives an `ok: true`
  result.

## Security Requirements

- The extension must connect only to `http://mcp.brosdk.internal`.
- BroSDK Browser owns the internal-host routing to the actual local MCP service.
- The extension must not send cookies, page DOM, local storage, proxy secrets, or
  fingerprint configuration values through this protocol.
- Server implementations should reject requests from unexpected origins when the
  runtime provides origin information.
- Command payloads must be validated before execution.

## Versioning

The current protocol version is `1.0.0`.

Backward-compatible additions may add optional fields, new command types, or new
capabilities. Breaking changes require a new major version and should be
advertised in `protocolVersion`.
